# RiseUP — iOS

The Xcode project (`Runner.xcodeproj/project.pbxproj`) is generated by Flutter
on first build. **Don't try to hand-edit it.** Instead:

## One-time setup

From the project root (`app/`):

```bash
flutter create --org com.albertlaudia --project-name riseup --platforms=ios .
```

This regenerates the missing pieces:
- `ios/Runner.xcodeproj/project.pbxproj`
- `ios/Runner.xcworkspace/contents.xcworkspacedata`
- `ios/Flutter/Generated.xcconfig` (overwrites the template)
- `ios/Runner.xcodeproj/xcshareddata/xcschemes/Runner.xcscheme`

Then:

```bash
cd ios
pod install
cd ..
flutter run -d "iPhone 15"
```

## Why we keep these files committed

Everything in this directory that's NOT in `.gitignore` is either:
- Apple-format binary (storyboards, plists) — needed for Xcode to build.
- The Podfile — defines iOS dependencies for CocoaPods.
- `Info.plist` — has all our permission strings + deep-link scheme.

If you `flutter create .` on a fresh clone, these files get overwritten.
**That's fine** — but if you've added custom permission strings or URL
schemes to `Info.plist`, keep a backup.

## Bundle identifier

`com.albertlaudia.riseup` — locked across:
- `ios/Runner/Info.plist` → CFBundleIdentifier via $(PRODUCT_BUNDLE_IDENTIFIER)
- Xcode → set in the project file
- `android/app/build.gradle` → applicationId

Don't change one without changing all three.

## Universal Links (deep link upgrade)

The current scheme is `riseup://library/{slug}` (custom URL scheme).
For App Store-quality deep linking, set up Universal Links:

1. Add an `apple-app-site-association` JSON to `https://riseup.app/.well-known/`
2. Add the `applinks:` entitlement to `ios/Runner/Runner.entitlements`
3. Wire it through Xcode → Signing & Capabilities → Associated Domains

Defer until you have a hosted landing page. The custom-scheme fallback
works for now.
