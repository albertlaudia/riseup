# RiseUP — TWA (Trusted Web Activity) setup

The web app is live at:

```
https://xyc4pio8o5le.space.minimax.io
```

It already has everything a TWA needs: HTTPS, a `manifest.json`, app icons
(192/512), a theme color, and a placeholder `/well-known/assetlinks.json`
that Bubblewrap will fill in with your signing key fingerprint.

## One-time setup (your machine)

1. **Install Flutter? No — install just these:**
   - **Node.js 18+** (you likely already have it)
   - **Java JDK 17** — `brew install openjdk@17` (mac) or `apt install openjdk-17-jdk` (linux)
   - **Android Studio** (or just the Android SDK + `platform-tools`) — for the SDK and to sign AABs

2. **Install Bubblewrap:**
   ```bash
   npm install -g @bubblewrap/cli
   bubblewrap doctor    # checks your toolchain
   ```

3. **Sign up for a Google Play Developer account** (one-time, $25): https://play.google.com/console
   - Verify your identity (takes a few days)

## Build the TWA

```bash
mkdir riseup-twa && cd riseup-twa
bubblewrap init --manifest=https://xyc4pio8o5le.space.minimax.io/manifest.json
```

Bubblewrap will:
- Read the manifest from your live URL
- Ask for your Android **package name** (suggested: `com.albertlaudia.riseup` or `app.riseup.stoic`)
- Ask for your **app name** (RiseUP)
- Ask for your **keystore password** — first time only, creates `android.keystore` in this folder
- Generate `twa-manifest.json` + the full Android project

When it asks, you can accept all defaults. When it's done:

```bash
bubblewrap build
```

This produces:
- `app-release-bundle.aab` — upload to Play Store
- `app-release-signed.apk` — for sideloading / local testing

## Push to Play Store

1. Go to https://play.google.com/console
2. **Create app** → choose "App" (not Game), name: "RiseUP — Stoic Practice"
3. Fill in store listing (screenshots, descriptions, category: Health & Fitness or Education)
4. Set up **Internal testing** track first — least friction
5. Upload your `app-release-bundle.aab`
6. **Service account** (one-time):
   - Google Cloud Console → create service account with "Play Android Developer" role
   - Download JSON key
   - Play Console → Setup → API access → link the service account
7. Subsequent uploads: just `bubblewrap build` + upload the new AAB

## What Bubblewrap also does (you don't need to touch it)

- Fills in `/well-known/assetlinks.json` on the host with your SHA-256 fingerprint
- This proves to Android that the URL belongs to your app — without it, the TWA shows up as "Powered by Chrome" instead of your app name

## Updating content

The web app is fully static. To push new lessons or quotes:
1. Edit / re-seed the PocketBase `rup_*` collections (`/workspace/riseup/scripts/seed.mjs`)
2. Rebuild: `cd /workspace/riseup/web && npx next build`
3. Re-deploy the `out/` directory to the same URL — **or** just point the TWA at a new domain

The TWA itself doesn't need to change unless you ship new app features
(native splash, push notifications, etc.).

## Optional: FCM push notifications

For daily reminders, you'll want Firebase Cloud Messaging:
- Native Android `FirebaseMessagingService` reads FCM and posts a `NotificationCompat.Builder` notification
- This is added on top of the bubblewrap-generated Android project, not generated by bubblewrap
- Happy to scaffold the FCM service class + manifest entries + Play Console FCM setup when you're ready

## If you hit snags

- **`bubblewrap doctor` fails on Java** — install JDK 17, set `JAVA_HOME`
- **Build fails on missing SDK** — open Android Studio once, finish the SDK setup wizard, then retry
- **Asset links verification fails on device** — wait 5-10 min after first install (Google's indexer caches)
- **App shows "Powered by Chrome" instead of your name** — means asset links didn't verify, check `/.well-known/assetlinks.json` matches the keystore fingerprint
